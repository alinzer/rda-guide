This guide shows how to authenticate, submit feeds, and retrieve results using the RDA Noteworthy API.

1. Authentication

Use your Client ID and Client Secret to obtain an access token. This token must be included in the Authorization header for all subsequent API requests.

curl
Python
export CLIENT_ID="your_client_id"
export CLIENT_SECRET="your_client_secret"

ACCESS_TOKEN=$(curl -s -X POST https://chemed-dev.rdanoteworthy.com/api/v1/auth/token \
  -H "Authorization: Basic $(echo -n $CLIENT_ID:$CLIENT_SECRET | base64)" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials" | jq -r '.access_token')
  
echo "Access Token: $ACCESS_TOKEN"
import base64
import requests

CLIENT_ID = "your_client_id"
CLIENT_SECRET = "your_client_secret"
auth_header = base64.b64encode(f"{CLIENT_ID}:{CLIENT_SECRET}".encode()).decode()

response = requests.post(
    "https://chemed-dev.rdanoteworthy.com/api/v1/auth/token",
    headers={
        "Authorization": f"Basic {auth_header}",
        "Content-Type": "application/x-www-form-urlencoded",
    },
    data={"grant_type": "client_credentials"},
)
data = response.json()
ACCESS_TOKEN = data["access_token"]
print("Access Token:", ACCESS_TOKEN)

2. Submit a Feed

Once you have an access token, submit a text file feed for processing. Currently, only the notes feed is implemented.

curl
Python
export COBDATE="20251105"
export FEED="notes"

curl -X POST \
  "https://chemed-dev.rdanoteworthy.com/api/v1/feeds/$COBDATE/$FEED/submit" \
  -H "Authorization: Bearer $ACCESS_TOKEN" \
  -H "Content-Type: multipart/form-data" \
  -F "file=@notes.csv;type=text/csv"
import requests
        
files = {"file": ("notes.csv", open("notes.csv", "rb"), "text/csv")}
cobdate = 20251105
feed = "notes"

response = requests.post(
    "https://chemed-dev.rdanoteworthy.com/api/v1/feeds/{cobdate}/{feed}/submit",
    headers={"Authorization": f"Bearer {ACCESS_TOKEN}"},
    files=files,
)
print(response.status_code, response.text)

3. Download Daily Updates

Retrieve the processed report for a given date. This includes all new and updated records from relevant feed(s) submitted on that date. Historical data from prior dates is not included. Currently, only the note-review report is implemented.

curl
Python
export COBDATE="20251105"
export REPORT="note-review"

curl -X GET \
  "https://chemed-dev.rdanoteworthy.com/api/v1/downloads/$REPORT/$COBDATE/delta" \
  -H "Authorization: Bearer $ACCESS_TOKEN"
import requests

cobdate = 20251105
report = "note-review"
response = requests.get(
    "https://chemed-dev.rdanoteworthy.com/api/v1/downloads/{report}/{cobdate}/delta",
    headers={"Authorization": f"Bearer {ACCESS_TOKEN}"},
)
print(response.status_code)
print(response.text)
